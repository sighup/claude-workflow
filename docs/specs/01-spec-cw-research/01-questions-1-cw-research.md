# Clarifying Questions — Round 1

## Q1: Interaction Model
**Question:** Should cw-research be interactive or autonomous?
**Answer:** Hybrid with phases — auto-explore first, then ask user to confirm/refine focus areas before deep-diving.

## Q2: Output Format
**Question:** Should output pipe directly into cw-spec or be standalone?
**Answer:** Both — save a structured report AND generate a pre-filled /cw-spec meta-prompt enriched with findings.

## Q3: Research Dimensions
**Question:** Which dimensions to cover?
**Answer:** All five: Architecture & patterns, Dependencies & integrations, Test & quality patterns, Data models & API surface, External context sources.

## Q4: Agent Definition
**Question:** Create agents/researcher.md for team workflows?
**Answer:** Yes.

## Q5: External Context Sources (follow-up)
**Question:** What does "External Context Sources" mean?
**Answer:** User-directed context from outside the repo — GitHub issues, Jira, Confluence, Google Docs, filesystem, images, web searches, documentation, knowledgebase articles, architecture decision records, or any other external content the user points to.
